2. ping 8.8.8.8

The ping command sends ICMP Echo Request packets to a target host to test network connectivity.

The IP address 8.8.8.8 belongs to Google Public DNS and is commonly used because it is reliable and almost always reachable.

The objective of running this command is to verify that the Kali Linux VM has internet access and can communicate with external networks.

3. IP Addresses of Kali and Windows

The IP address of the Kali Linux VM was identified using the ip a command.

The IP address of the Windows host machine was identified using the ipconfig command.

A ping test was performed between the two machines.

Communication depends on the VirtualBox network mode used; it is usually not possible in NAT mode but works in Bridged mode.

4. VirtualBox Network Modes (NAT vs Bridged)

NAT mode allows the VM to access the internet but prevents direct communication with the host.

Bridged Adapter mode connects the VM directly to the same network as the host.

The VM is able to ping the host machine and vice versa when using Bridged Adapter mode.

This happens because both machines receive IP addresses from the same network and can communicate directly.

8. Accessing XAMPP from Kali Linux

An attempt was made to access the XAMPP web server from Kali Linux using the Windows host IP address.

Access was successful only when the VM network mode was set to Bridged Adapter.

In NAT mode, access failed due to network isolation.

Disabling or allowing Apache through the Windows Firewall was necessary to allow external access.

Apache had to be running and both machines needed to be on the same network.
